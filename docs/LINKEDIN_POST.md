# LinkedIn Post Draft

I built a local-first Bangla speech dataset studio around Liquid models.

The project does not make unsupported audio-model claims. Instead, it audits the local LFM-Audio runner, evaluates a Bangla ASR baseline, measures Liquid text-model repair potential, applies abstaining evidence routing, and generates schema-valid reviewable action cards.

Key result:
- LFM-Audio runner audit completed, but ASR claims were blocked because no valid transcripts were produced.
- BanglaASR produced usable baseline transcripts.
- Liquid text repair showed evaluation-only correction potential.
- The router abstained when evidence was weak and reached high accuracy on covered examples.
- Final action cards were 100% JSON-valid.

This is designed as a community project for practical, claim-safe local AI workflows.
